# S1 Assistant Architecture

## System Overview
S1 is a premium, human-like virtual assistant designed for desktop environments. It features a unified brain pipeline, multi-language support, and a modern animated interface.

## Core Pipeline
1.  **Input:** Voice (Wake Word "Hey S1") or Text Entry.
2.  **Processing (MasterBrainV7):**
    *   **Emotion Detection:** Analyzes tone (Happy, Angry, etc.).
    *   **NLP:** Multi-task expansion and intent mapping.
    *   **Context:** Resolves "it/this" using short-term history.
    *   **Memory:** Recalls personal facts and preferences.
    *   **Decision:** Risk-aware strategy (Execute, Ask, Refuse).
3.  **Action:** Executed via `ActionEngine` with `ActionGuard` security.
4.  **Response:** Human-like reply generated by `ResponseEngine` with emotional adaptive tone.
5.  **Output:** Animated UI (Orb) and Voice (TTS).

## Key Modules
- **`core/master_brain_v7.py`**: The central orchestrator.
- **`core/action_engine.py`**: The unified skill executor.
- **`ui/premium_app.py`**: Modern CustomTkinter interface.
- **`voice/wake_word.py`**: Background keyword listener.
- **`security/action_guard.py`**: Hardened security layer.

## State Management
S1 uses a Global State Manager with the following states:
- `IDLE`: Waiting for Wake Word.
- `LISTENING`: Actively capturing audio.
- `THINKING`: Processing command.
- `SPEAKING`: Outputting response.
- `WAITING`: Brief window for follow-up commands (7s).
